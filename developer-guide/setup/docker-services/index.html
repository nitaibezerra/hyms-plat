    Docker Services¶

    Serviços externos gerenciados via Docker.

    Visão Geral¶

    O projeto usa três serviços principais:

    Serviço Imagem Porta
    PostgreSQL postgres:16-alpine 5432
    Redis redis:7-alpine 6379
    TypeSense typesense/typesense:27.1 8108

    Docker Compose¶

    # docker-compose.yml
version: '3.8'

services:
  postgres:
    image: postgres:16-alpine
    environment:
      POSTGRES_DB: hymnplat
      POSTGRES_USER: hymnplat
      POSTGRES_PASSWORD: hymnplat
    ports:
      - "5432:5432"
    volumes:
      - postgres_data:/var/lib/postgresql/data
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U hymnplat"]
      interval: 10s
      timeout: 5s
      retries: 5

  redis:
    image: redis:7-alpine
    ports:
      - "6379:6379"
    volumes:
      - redis_data:/data
    healthcheck:
      test: ["CMD", "redis-cli", "ping"]
      interval: 10s
      timeout: 5s
      retries: 5

  typesense:
    image: typesense/typesense:27.1
    ports:
      - "8108:8108"
    environment:
      TYPESENSE_API_KEY: xyz
      TYPESENSE_DATA_DIR: /data
    volumes:
      - typesense_data:/data

volumes:
  postgres_data:
  redis_data:
  typesense_data:

    Comandos¶

    Iniciar Todos¶

    docker-compose up -d

    Parar Todos¶

    docker-compose down

    Ver Logs¶

    # Todos os serviços
docker-compose logs -f

# Serviço específico
docker-compose logs -f postgres

    Reiniciar Serviço¶

    docker-compose restart postgres

    Limpar Dados¶

    # Para e remove volumes
docker-compose down -v

    PostgreSQL¶

    Conexão¶

    # Via psql
docker-compose exec postgres psql -U hymnplat

# Via Django
poetry run python manage.py dbshell

    Backup¶

    docker-compose exec postgres pg_dump -U hymnplat hymnplat > backup.sql

    Restore¶

    cat backup.sql | docker-compose exec -T postgres psql -U hymnplat

    Redis¶

    Conexão¶

    docker-compose exec redis redis-cli

    Comandos Úteis¶

    # Ping
redis-cli ping

# Ver todas as keys
redis-cli keys '*'

# Limpar cache
redis-cli flushall

    TypeSense¶

    Health Check¶

    curl http://localhost:8108/health

    Ver Collections¶

    curl -H "X-TYPESENSE-API-KEY: xyz" \
  http://localhost:8108/collections

    Ver Estatísticas¶

    curl -H "X-TYPESENSE-API-KEY: xyz" \
  http://localhost:8108/collections/hymns

    Troubleshooting¶

    Container não inicia¶

    # Ver logs
docker-compose logs postgres

# Verificar recursos
docker stats

    Porta já em uso¶

    # Encontrar processo
lsof -i :5432

# Matar processo
kill -9 <PID>

# Ou mude a porta no docker-compose.yml
ports:
  - "5433:5432"

    Dados persistentes¶

    Os dados são persistidos em Docker volumes:

    # Listar volumes
docker volume ls

# Inspecionar volume
docker volume inspect hyms-plat_postgres_data
